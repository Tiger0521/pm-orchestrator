## 一、部门定位与职责

### 1.1 部门属性

- 所属 ：联通的后域网络设备管理部门

- 管辖范围 ：所有入网的网络设备

- 涵盖 ：网络运营事业部的相关工作

### 1.2 具体工作内容

#### 设备监控

入网设备的运行状态监控

#### 资源调拨

网络资源的调配与管理

#### 跨部门协调

跨部门沟通协调

#### 业务覆盖

网络连接、数据承载等领域

### 1.3 中层管理职责

- **保障网络连接**

- **联通相关系统的开发建设**

## 二、网络资源管理范畴

### 2.1 核心资源类型

| 资源类型 | 说明 | 特点 |
| --- | --- | --- |
| 设备 | 入网网络设备 | 需全生命周期管理 |
| 光缆（管线资源） | 通信光缆 | 典型无源资源，只需铺设不需额外供电 |
| 基站（无线资源） | 由一组设备组成的功能单元 | 典型特点是可测算 |

### 2.2 无线资源详解

**重要概念**：没有通常意义上的单个基站，基站是**功能单元**，由多个设备组成。

5G基站可能包含以下组件：

- **天线**：包含有源和无源天线

- **CU**（Central Unit，中央单元）

- **AU**（Access Unit，接入单元）

各类无线相关设备均需要进行管理。

### 2.3 其他需管理的资源

| 资源类型 | 说明 |
| --- | --- |
| 空间资源 | 联通机房，需记录面积、位置等信息 |
| 行政区域数据 | 需维护的行政区域相关数据 |

### 2.4 机房信息管理

| 信息项 | 说明 |
| --- | --- |
| 机房面积 | 需询问并记录 |
| 机房位置 | 如某栋楼的10层 |
| 录入方式 | 在系统中手动录入 |

## 三、资源管理理念

### 3.1 资产到资源的转变

投入资金后，资产转变为资源，这些资源后续需要运维运行。

### 3.2 运维运营工作

- 运营整个网络

- 保障网络安全

- 确保资源处于可用状态

目前处于可用状态的资源，都需要我们进行管理。

## 四、基础维护与数据治理

### 4.1 基础维护机制

**人工保障**：即使采集解析的数据有误或更新不及时，也可以人工进行修改。

### 4.2 数据治理流程

- **合规性检查**：对入库数据进行合规性检查

- **比对流程**：正常情况下自动比对，但网络配置等内容也可人工操作

### 4.3 比对后的两种情况

| 类型 | 说明 |
| --- | --- |
| 资源待办 | 比对发现异常（如新增/删除设备），需人工核实真实性和准确性。例如：比对后发现昨天没有采集到某台重要的广元级设备的文件，今天该设备出现，需要人工核实这台设备是否真的接入网络。 |
| 全生命周期管理 | 设备从规划到入网及后续运维的全流程管理。新增或删除设备时，都需要人工核实数据的真实性和准确性。 |

#### 全生命周期管理详细流程

规划 → 项目立项 → 施工 → 采购 → 正式接入网络

#### 后续运维环节

#### 设备更新

#### 零部件更换

#### 设备搬迁

#### 软件更新

#### 业务割接

#### 扩容

**核心原则**：每种流程都会影响设备数据情况（如扩容后内存从8G变为16G），必须走流程，**不能随意修改**。

## 五、故障排查场景

### 5.1 典型场景

通信链路中存在很多环节，当双方电话打不通时，需要尽快定位故障原因。

### 5.2 排查目标

- 定位是否为**光缆被挖断**导致

- 定位是否为**设备端口故障**导致

### 5.3 数据依赖

故障排查的各个环节都离不开**网络资源相关数据**的支持。

## 六、数据共享方式

| 方式 | 说明 |
| --- | --- |
| 接口服务 | 提供可随时查询、调用的接口 |
| 只读数据库 | 提供只读数据库，可直接在库中查询分析 |
| 文件共享 | 将数据做成文件，定期上传至类似百度网盘的存储位置，供使用者每日下载后自行分析 |

## 七、工作目标（三层级）

| 层级 | 目标 |
| --- | --- |
| 第一层 | 保证数据完整 — 所有数据都应在系统中留存 |
| 第二层 | 保证数据准确 |
| 第三层 | 场景化应用支持 — 可定制化开发服务或应用，让用户在特定场景下使用数据 |

整体工作目标是让各个环节的数据变更与现实情况保持一致，核心是做好数据维护。

## 八、数据更新与查询痛点

| 痛点 | 说明 | 解决方向 |
| --- | --- | --- |
| 数据更新繁琐 | 用户不愿手动逐行填写大量字段 | 实现快速的数据更新 |
| 数据查询困难 | 多表嵌套查询复杂，一线用户不懂技术，往往只知道设备名称，想要快速查询设备端口数量、端口占用情况以及设备间的连接关系等信息 | 自然语言查询优化，理想状态是用户一句话完成查询 |

**现状**：目前自然语言查询功能还不完善，需通过功能优化解决。一线用户大多不懂计算机技术，往往只知道设备名称。

## 九、系统权限与后续安排

- **权限开通**：后续将开通几个简单权限，不涉及复杂操作

- **资源查看**：开通后可查看无线资源和管线资源

- **账号处理**：等账号开通后，由项目经理处理相关内容
