# 需求分析阶段质量门

阶段完成并推进到“需求拆解”前，必须逐条通过以下校验。

---

## 文件存在性

- [ ] `docs/strategic/req-001.md` 存在
- [ ] `docs/strategic/epic-001.md` 存在
- [ ] 至少一个 `docs/requirement/feature-*.md` 存在

---

## Frontmatter 完整性

每份文档必须包含：

- [ ] `id`
- [ ] `type`
- [ ] `projectId`
- [ ] `title`
- [ ] `status`
- [ ] `refs`（Epic、Feature 至少一条引用关系；需求卡片可为空数组）

---

## 诊断报告质量

- [ ] 用户已确认诊断报告
- [ ] 诊断报告包含问题本质还原
- [ ] 诊断报告包含需求转化记录
- [ ] 诊断报告包含关键假设和待验证事项
- [ ] 诊断报告包含项目类型判断（`new | iteration | refactor`）
- [ ] 诊断报告包含需求成熟度评分（0-10 分）
- [ ] 评分依据清晰：Q1-Q7 回答质量、数据支撑、用户画像、投入意愿
- [ ] 低于 5 分时，`tracking-log.md` 已记录补充调研计划，且未进入正式文档产出
- [ ] 替代方案不少于 2 个，且包含成本、时间、风险、ROI、适用条件

---

## 需求卡片质量

- [ ] 问题本质描述清晰，不超过 3 句话
- [ ] 目标用户画像具体，包含核心用户、典型场景、痛点强度、切换阈值
- [ ] 当前替代方案及痛点已说明
- [ ] 需求转化记录已说明原始表述、识别问题、转化后需求、转化依据
- [ ] 关键假设已列出（至少 1 条）
- [ ] 成功指标可量化（1-3 个）
- [ ] 需求评估结果三维度齐全：价值、紧急度、可行性
- [ ] 每个评估维度都有理由
- [ ] 待验证事项已列出或明确无待验证项

---

## Epic 质量

- [ ] 需求背景引用了需求卡片
- [ ] 产品名称已填写
- [ ] 产品定位一句话说清
- [ ] 产品目标可量化
- [ ] 战略价值明确
- [ ] 目标用户/角色已列出
- [ ] 核心场景至少 1 个
- [ ] 边界清晰，明确范围内和范围外
- [ ] 建设思路包含分阶段路径
- [ ] 成功指标可量化

---

## Feature 质量

- [ ] 需求背景引用了需求卡片
- [ ] 能力名称已填写
- [ ] 能力描述清晰
- [ ] 能力目标可量化
- [ ] 用户角色引用了 Epic
- [ ] 业务场景已列出（1-3 个）
- [ ] 业务价值说明充分
- [ ] 业务流程已描述
- [ ] 业务规则已列出
- [ ] 技术可行性已评估
- [ ] 资源投入已估算（人天、人周或依赖资源）
- [ ] 优先级已标注（P0/P1/P2）
- [ ] 依赖关系已说明
- [ ] 验收标准可验证

---

## 数据校验

- [ ] 所有写入 `facts.json` 的数据已标注来源类型：`manual-input`、`file-extract`、`web-reference` 或 `derived-and-confirmed`
- [ ] 文件提取的数据经过格式校验和范围校验
- [ ] 数字口径与文档表述一致
- [ ] 校验异常数据已隔离，且在诊断报告“数据校验异常”章节列出
- [ ] 二手数据已标注来源和时间，经用户确认后才采纳
- [ ] 无来源标注的数据未被采纳

---

## 对抗性自审

五维度评分必须均不低于 7 分：

- [ ] 完整性：关键字段、引用关系、诊断链条齐全
- [ ] 一致性：需求卡片、Epic、Feature 之间无矛盾
- [ ] 清晰度：用户、场景、问题、目标可理解
- [ ] 范围：最小切入点明确，无平台化范围膨胀
- [ ] 可行性：技术、数据、资源、流程前提已识别
- [ ] 无未解决的 `validation-failed` 项
- [ ] 迭代轮次不超过 3 轮

---

## 用户确认

- [ ] 用户已确认诊断报告（含需求转化记录和成熟度评分）
- [ ] 用户已确认数据校验异常项（逐条核实或修正）
- [ ] 用户已从替代方案中选择主方案
- [ ] 用户已确认需求卡片、Epic 和 Feature 内容

---

## 记忆更新

- [ ] `refs.json` 已注册所有新文档节点和引用边
- [ ] `facts.json` 已记录已确认事实，且每条事实有来源
- [ ] `decision-log.md` 已记录方案选择、理由和被否定方案
- [ ] `tracking-log.md` 已记录未验证假设、风险、未决问题
- [ ] `phase-summary.md` 已追加本阶段摘要
- [ ] `progress.json` 已更新文档列表和阶段内进度
- [ ] `progress.json.currentPhase` 未被 subagent 修改
