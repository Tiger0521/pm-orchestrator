# 需求分析阶段质量门

阶段完成并推进到「需求拆解」前，必须逐条通过以下校验。

---

## 文件存在性

- [ ] `docs/strategic/req-001.md` 存在
- [ ] `docs/strategic/epic-001.md` 存在
- [ ] 至少一个 `docs/requirement/feature-*.md` 存在

---

## Frontmatter 完整性

每份文档必须包含：

- [ ] `id`
- [ ] `type`
- [ ] `projectId`
- [ ] `title`
- [ ] `status`
- [ ] `refs`（至少一条引用关系）

---

## 需求卡片质量

- [ ] 问题本质描述清晰，不超过 3 句话
- [ ] 目标用户画像具体（角色、场景、痛点）
- [ ] 当前替代方案及痛点已说明
- [ ] 关键假设已列出（≥1 条）
- [ ] 成功指标可量化（1-3 个）

---

## Epic 质量

- [ ] 产品定位一句话说清
- [ ] 战略价值明确
- [ ] 目标用户/角色已列出
- [ ] 核心场景 ≥1 个
- [ ] 边界清晰（明确不做哪些事）
- [ ] 成功指标可量化

---

## Feature 质量

- [ ] 每个 Feature 有清晰描述
- [ ] 用户价值说明充分
- [ ] 业务流程已描述
- [ ] 业务规则已列出
- [ ] 优先级已标注（P0/P1/P2）
- [ ] 依赖关系已说明

---

## 用户确认

- [ ] 用户已确认诊断报告
- [ ] 用户已从替代方案中选择主方案
- [ ] 用户已确认 Epic 和 Feature 内容

---

## 记忆更新

- [ ] `progress.json` 已更新阶段状态
- [ ] `refs.json` 已注册所有新文档节点和引用边
- [ ] `facts.json` 已记录已确认事实
- [ ] `decision-log.md` 已记录方案选择及理由
- [ ] `tracking-log.md` 已记录未验证假设/风险/未决问题
- [ ] `phase-summary.md` 已追加本阶段摘要
